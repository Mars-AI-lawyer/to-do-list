### 修复毛玻璃圆球不可见问题

**问题**：macOS 上 `transparent: true` + CSS `backdrop-filter` 组合无法正常渲染毛玻璃圆球。原因是透明 Electron 窗口在合成器中没有"背后内容"可供 CSS backdrop-filter 模糊，导致圆球几乎不可见。

**修复内容** (`src/main/window/mainWindow.ts`):
- 添加 `vibrancy: 'popover'` — 使用 macOS 原生 NSVisualEffectView 实现毛玻璃效果
- 添加 `visualEffectState: 'active'` — 确保毛玻璃效果始终激活
- 修改 `hasShadow: false` → `hasShadow: true` — 恢复窗口阴影增强可见性
- 添加 `backgroundColor: '#00000000'` — 确保透明合成正确

**修复内容** (`src/renderer/components/Bubble.vue`):
- 圆球背景不透明度 0.72 → 0.88（增强可见性）
- 阴影加深，增加视觉层次

构建验证通过。

### 第二次修复：vibrancy 与 transparent 冲突

**问题**：`transparent: true` + `vibrancy` 在 Electron 中不兼容。vibrancy 需要窗口有实际背景层才能渲染 NSVisualEffectView，transparent 会阻止该层创建。

**修复内容** (`src/main/window/mainWindow.ts`):
- 移除 `vibrancy: 'popover'`、`visualEffectState: 'active'`、`backgroundColor: '#00000000'`
- 仅保留 `transparent: true`，让窗口保持透明

**修复内容** (`src/renderer/components/Bubble.vue`):
- 移除 `backdrop-filter` / `-webkit-backdrop-filter`（透明窗口无法模糊桌面内容）
- 改用纯 CSS 模拟玻璃质感：
  - 径向渐变 `radial-gradient` 模拟玻璃高光（中心亮、边缘淡蓝）
  - 多层 `box-shadow`：外阴影营造悬浮感 + 内阴影营造厚度 + 1px 描边
  - 边框使用半透明白色
- hover 状态同步更新，保持一致的视觉语言

**原理**：在 56×56 像素的极小窗口上，真正的"背景内容模糊"即使生效也几乎不可见。改用精心调制的渐变+阴影组合，在任何桌面背景下都能呈现出精致的玻璃质感，且不受 Electron 透明窗口合成限制。

构建验证通过。

### 第三次修复：全新视觉 + 拖拽功能

**问题**：
1. 径向渐变+阴影的玻璃模拟效果仍不理想，用户要求"其他效果"
2. 原有的 `-webkit-app-region: drag` 拖拽区域过小（仅顶部 20×8px 小条），用户无法找到拖拽点

**修复内容** (`src/renderer/components/Bubble.vue` 完全重写):
- **全新视觉风格**：放弃"毛玻璃"思路，改用干净的白底 + 蓝色描边阴影方案
  - 背景：`linear-gradient(180deg, #ffffff, #f8f9fc, #f0f2f7)` — 纯白到极淡蓝灰，任何背景下都有清晰轮廓
  - 阴影：蓝色主调阴影 `0 6px 20px rgba(0,122,255,0.12)` + 深色辅助阴影 + 1px 蓝色描边
  - hover 状态：放大 1.1 倍，描边和阴影加深，颜色变深
- **自定义拖拽实现**（替代失效的 CSS drag 区域）：
  - `mousedown` 记录起始位置
  - `mousemove` 计算移动增量，通过 IPC `window:drag-move` 发送给主进程
  - 移动超过 3px 阈值即进入拖拽模式
  - `mouseup` 时判断：若未进入拖拽模式则触发点击展开
  - 使用 `@mousedown.prevent` 避免选中文本等副作用

**修复内容** (`src/main/window/mainWindow.ts`):
- 新增 `WINDOW_DRAG_MOVE` IPC 处理器：接收 deltaX/deltaY，调用 `setBounds()` 实时更新窗口位置

**修复内容** (`src/preload/index.ts` + `src/shared/ipc-channels.ts` + `src/renderer/env.d.ts`):
- 新增 `window:drag-move` IPC 通道和 `dragMove(deltaX, deltaY)` API

构建验证通过。
